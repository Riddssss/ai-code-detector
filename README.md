## 🎯 Problem Statement

As AI-generated code becomes increasingly indistinguishable from human-written code, there is a growing need for automated tools to detect its origin. This project proposes a system that uses **character-level n-gram TF-IDF features** combined with **hand-crafted statistical features** to classify code snippets as human or AI-authored.

---

## 🏗️ System Architecture

```
AI PDF + Human PDF
        ↓
  PDF Text Extraction (pdfplumber / PyPDF2)
        ↓
  Code Snippet Splitting (heuristics)
        ↓
  Preprocessing & Cleaning
        ↓
  ┌─────────────────────────────────┐
  │  Feature Extraction             │
  │  ├── TF-IDF char n-grams (3–6) │
  │  └── Statistical Features (7)  │
  └─────────────────────────────────┘
        ↓
  ┌──────────────────────────┐
  │  Model Training          │
  │  ├── Logistic Regression │
  │  └── Random Forest       │
  └──────────────────────────┘
        ↓
   Flask REST API + Gradio UI
```

## ✨ Features

### Core Pipeline
- 📄 **PDF Ingestion** — extracts code snippets from AI and human code PDFs
- 🔤 **TF-IDF Character N-grams** — captures coding style at the character level (3–6 grams, 20k features)
- 📐 **Statistical Features** — 7 hand-crafted signals: comment ratio, token count, avg line length, imports, definitions, special symbols, avg token length
- 🤖 **Dual Classifiers** — Logistic Regression (fast) and Random Forest (robust)

### User Features
- ⚡ **Gradio UI** — paste any code snippet and get an instant prediction
- 🌐 **Flask REST API** — `/predict` endpoint for programmatic access
- 🔀 **Model Toggle** — switch between Logistic Regression and Random Forest at inference time
- 📊 **Probability Output** — returns confidence scores for both classes

## 🧠 Algorithm Details

### Feature Representation

```
Final Feature Matrix = TF-IDF (char n-grams) ⊕ Statistical Features
                       [20,000 dims]              [7 dims]
```

### Statistical Features

| Feature | Description |
|---------|-------------|
| `avg_line_len` | Average characters per line |
| `comment_ratio` | Fraction of lines that are comments |
| `num_imports` | Count of import / #include statements |
| `num_defs` | Count of def / class / function keywords |
| `num_special_syms` | Count of special symbols `{}()<>;` etc. |
| `token_count` | Total word/token count |
| `avg_token_len` | Average token length |

### TF-IDF Config

| Parameter | Value |
|-----------|-------|
| Analyzer | `char_wb` (character n-grams within word boundary) |
| N-gram range | (3, 6) |
| Max features | 20,000 |

### Model Config

| Parameter | Logistic Regression | Random Forest |
|-----------|--------------------|--------------:|
| Max iterations | 2000 | — |
| Estimators | — | 200 |
| Train/Test split | 80 / 20 | 80 / 20 |
| Random state | 42 | 42 |

## 🚀 How to Run

### Prerequisites
- Python 3.12+

### Step 1 — Install dependencies
```bash
cd backend
pip install -r requirements.txt
pip install gradio
```

### Step 2 — Train the models
```bash
cd backend
python train.py --ai_pdf "ai codes.pdf" --human_pdf "human code.pdf"
```

This produces:
- `code_dataset.csv`
- `tfidf_vectorizer.joblib`
- `scaler.joblib`
- `trained_models/logreg_model.joblib`
- `trained_models/rf_model.joblib`

Already have `code_dataset.csv`? Skip PDF extraction:
```bash
python train.py --skip_pdf
```

### Step 3 — Launch Gradio UI
```bash
python gradio_app.py
```
Open: `http://127.0.0.1:7860`

### Step 4 — (Optional) Launch Flask API
```bash
cd backend
python app.py
```
API available at: `http://127.0.0.1:5000`

## 🔌 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/health` | Health check |
| POST | `/predict` | Classify a code snippet |
| GET | `/predict?code=...&model=logreg` | Classify via query param |

### Example Request
```bash
curl -X POST http://127.0.0.1:5000/predict \
  -H "Content-Type: application/json" \
  -d '{"code": "def hello():\n    print(\"Hello\")", "model": "logreg"}'
```

### Example Response
```json
{
  "label": "human",
  "prob_ai": 0.1234,
  "prob_human": 0.8766,
  "model_used": "logistic_regression"
}
```
